Quick Start

Quick Start

このサイトはgithubが提供している無料サービスを活用して公開しています。
個人事業のスタートアップでは、github上のサイトだけで済ませている例もまま有りますので、このサンプルページをそのまま使用することも出来なくもありません。

最終的には独自ドメインを用意して固有のサイトを持たれる方がよろしいかと存じます。
その際の切り分け方には二通りが御座います。

業者に管理させる場合

  • CHANCE側
    • 原稿、画像などの素材作成と提供
    • レイアウトの大まかな指示
  • 業者側
    • ドメインの取得管理
    • paasの選定、不具合対応
    • 原稿の形式変換
    • ページレイアウト
    • デプロイ

ご自身で編集管理される場合

  • 初度業務
    • ドメインの取得管理 （一度決めてしまえば、あとは年使用額の引き落しだけです。）
    • 静的サイト用paasの選定（一度決めてしまえば、あとは月使用額の引き落しだけです。静的サイト運用ですので滅多に障害は起きません。）
  • 都度業務
    • デプロイ （githubからpaasへのデプロイを管理者側が行う）
  • 作成担当
    • マークダウン形式での作成を義務づければ、原稿作成者が概ねのレイアウトを決め、形式変換作業は省略出来ます。

以下、概要を御説明いたします。

環境構築

  • お使いのPCにnodejsをインストールしておきます。
  • 任意のフォルダにhexo（静的サイトジェネレーター）をインストールします。
  • 上記のフォルダにgithubのCHANCEのリポジトリをクローンします。

それぞれweb上に詳しい手順と解説がいろいろとあります。用語に慣れないだけで、実際の学習コストは低いです。

githubを活用した共同編集

メンバーがhexoとgithubの使い方を覚えると、各自が個別に原稿を作成して投稿したり、共同編集できたりと、管理運営の手間が軽減されます。

最終的にpaasに独自ドメインを持って正式なサイトを発足させる場合でも、gituhub側を原稿編集とプレビューの場として活用する事が可能です。

  • 原稿作成者： hexoで原案を作成してgithubへデプロイ
  • 管理者や他のメンバー： githubのページを見て査読、監修、校正
  • 管理者： 完成稿をpaasにデプロイ（一般公開）

新規投稿

hexo new <投稿タイトル> で/source/_posts以下に原稿ファイルの雛形が作成されます。そのファイルをテキストエディタで開いて原稿を書き込んで行きます。原稿ファイルの形式はマークダウン形式です。

More info: Writing

ローカルサーバーの起動

御自身のPC上で更新や変更をプレビュー出来ます。ブラウザのhttp://localhost:4000でプレビューが出来ます。サーバー停止はCtrl+Cです。

More info: Server

花粉症（かふんしょう）

　みなさん、こんにちは。　お元気ですか？　春らしくなってきましたね。
　いい季節ですが、あることに悩まされる人が多いです。
それは花粉症です。　木や花の花粉でアレルギーを起こします。
症状は鼻水、くしゃみ、鼻づまり、目のかゆみ、涙、充血などです。
もし、今、あなたがこの症状に悩んでいるとしたら、病院へ行くことをお勧めします。　　市販の薬もありますが、きちんと診断してもらった方がいいです。
自分でできることは、マスク、めがね（ゴーグル）、ぼうしなどで身体に花粉がつかないようにすることです。↓
https://matome.naver.jp/odai/2139167228414367201/2139168977227826303
帰宅後にうがい、手洗い、洗顔をすることも有効です。
では、日本の春をお楽しみください。

Selamat menikmati musim semi.
